Getting Started

Use our getting_started/Lineup design system 3.0 with bootstrap. All frontend elements and style guidelines with Scss content are maintained as a npm package at the antifactory.

Use the command to add scss files and folder to the project.

npm install —save Rocket-lineup

Lineup styles are applied to the default elements such as input, select box and other common elements. The other elements such as buttons and status elements consist of several classes and guidelines that are explained in the lineup portal.

Lineup Functional Introduction

Theming in Lineup is accomplished by Sass variables, Sass maps, and custom CSS. There is no dedicated theme stylesheet; instead, you will be able to support the built-in theme to combine gradients, shadows, and so on.

Sass

Utilize our source Sass files related to variables, maps, mixins, and so on. In our build, we have increased the Sass rounding precision to 6 (by default it is 5) to avoid issues with browser rounding.

File structure

Whenever possible, avoid modifying Lineup core files. For Sass, it means creating your own stylesheet only for required panels colors or any other addon features. Assuming you are using a package manager like npm, the file structure looks like this:

your-project/
├── scss
│ └── custom.scss
└── node_modules/
└── @Rocketui/Rocketui
├── js
└── scss
